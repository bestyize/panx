# Panx 项目实现说明（给 Codex / 自动编码代理）

## 1. 任务目标

请基于当前仓库实现一个**可运行的文件网盘系统 MVP**。项目定位是：

- 面向个人或小团队的文件管理系统
- 支持文件夹、文件上传下载、重命名、移动、删除、搜索、分享链接
- 提供响应式 Web 界面，可在 PC 和移动端访问
- 后端提供 RESTful API，前端页面可直接使用服务端模板实现

> 重要原则：优先完成可运行、可测试、可演示的 MVP；不要一开始实现所有“高级功能”。

---

## 2. 当前项目技术基线

请**优先遵循当前仓库已有技术栈**，不要随意切换框架：

- 语言：Kotlin
- JDK：21
- 框架：Spring Boot 4.x
- Web：Spring Web MVC
- 模板引擎：Thymeleaf
- 数据访问：Spring Data JDBC
- 数据库：MySQL
- 文件存储：本地文件系统

如实现过程中确有必要，可以补充以下依赖：

- `spring-boot-starter-security`
- `spring-boot-starter-validation`
- `spring-boot-starter-actuator`
- 测试相关依赖

### 实现约束

1. **MVP 阶段不要引入 React / Vue / Angular 独立前端工程**。
2. 前端页面直接使用 `Thymeleaf + 原生 JavaScript + CSS` 实现。
3. 文件内容存储在本地目录，文件元数据存储在 MySQL。
4. 代码结构要清晰，按 controller / service / repository / domain / dto / config 分层。
5. 所有核心能力需要有基础测试覆盖。

## 3. 范围拆分

### 3.1 必须实现的 MVP 功能

#### 用户与认证

- 提供基础登录能力
- MVP 可先实现单管理员用户或基础账号体系
- 未登录用户不能访问管理后台
- 分享链接访问可匿名，但仅限被分享文件

#### 文件与文件夹管理

- 浏览文件夹内容
- 创建文件夹
- 上传单个/多个文件
- 下载文件
- 删除文件/文件夹
- 重命名文件/文件夹
- 移动文件/文件夹到其他目录
- 显示文件基础信息：名称、大小、类型、更新时间

#### 搜索

- 支持按文件名关键字搜索
- 支持在当前用户可见范围内搜索

#### 分享

- 为文件生成分享链接
- 分享链接可设置过期时间
- 分享链接可设置提取码（可选，但优先实现）
- 通过分享链接访问并下载文件

#### 前端页面

- 登录页
- 文件列表页
- 文件上传交互
- 新建文件夹、重命名、移动、删除交互
- 分享链接生成与展示
- 基础响应式布局，兼容手机和 PC

#### 基础预览

- 图片文件在线预览
- 文本文件在线预览
- 其他文件类型至少提供下载入口

## 4. 建议的系统设计

### 4.1 架构

系统采用单体架构：

- Web 层：页面路由 + REST API
- 业务层：文件管理、分享、搜索、认证
- 数据层：MySQL 保存元数据
- 存储层：本地文件系统保存实际文件

建议定义存储抽象接口，例如：

- `StorageService`
- `ShareService`
- `FileService`
- `AuthService`

这样后续替换为云存储时成本更低。

### 4.2 核心实体建议

至少包含以下实体：

#### User

- `id`
- `username`
- `passwordHash`
- `displayName`
- `enabled`
- `createdAt`
- `updatedAt`

#### FileNode

统一表示文件和文件夹：

- `id`
- `parentId`
- `ownerId`
- `name`
- `nodeType`（FILE / FOLDER）
- `storagePath`
- `contentType`
- `size`
- `extension`
- `sha256`（可选）
- `deleted`
- `createdAt`
- `updatedAt`

#### ShareLink

- `id`
- `fileNodeId`
- `shareToken`
- `extractCode`
- `expiresAt`
- `createdBy`
- `createdAt`
- `enabled`

#### AccessLog（可选）

- `id`
- `shareLinkId`
- `accessIp`
- `userAgent`
- `accessedAt`

## 5. 数据与存储规则

1. 数据库保存**文件元数据**，不要把二进制文件直接存数据库。
2. 文件实际内容保存到本地，例如：`data/storage/{userId}/{yyyy}/{MM}/...`
3. 删除文件时：
   - MVP 可以先做真实删除；
   - 如果实现回收站，则改为软删除。
4. 文件夹删除时必须校验：
   - 是否允许递归删除；
   - 是否包含子文件和子目录。
5. 文件重命名与移动时，要同步维护元数据，必要时同步调整物理路径。
6. 上传时应限制单文件大小、总请求大小、允许类型白名单或黑名单。

## 6. REST API 范围

请至少提供以下 API（可按实际情况微调路径，但要保持清晰一致）：

### 6.1 认证

- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/auth/me`

### 6.2 文件夹与文件

- `GET /api/files?parentId=`：获取目录列表
- `GET /api/files/tree`：获取目录树（可选简化版）
- `POST /api/files/folders`：创建文件夹
- `POST /api/files/upload`：上传文件（支持多文件）
- `GET /api/files/{id}/download`：下载文件
- `GET /api/files/{id}/preview`：预览文件
- `PATCH /api/files/{id}/rename`：重命名
- `PATCH /api/files/{id}/move`：移动
- `DELETE /api/files/{id}`：删除单个文件或文件夹
- `POST /api/files/batch-delete`：批量删除
- `GET /api/files/search?keyword=`：搜索

### 6.3 分享

- `POST /api/shares`：创建分享链接
- `GET /api/shares/{token}`：获取分享信息
- `POST /api/shares/{token}/verify`：校验提取码（如启用）
- `GET /api/shares/{token}/download`：通过分享链接下载
- `DELETE /api/shares/{id}`：取消分享

## 7. 页面与交互要求

### 管理后台页面

至少包含：

- 顶部导航栏
- 当前路径面包屑
- 文件列表区域
- 上传按钮
- 新建文件夹按钮
- 搜索框
- 批量选择操作区

### 交互要求

- 支持批量上传
- 支持批量删除
- 操作前要有明确反馈
- 异常情况要给出友好提示
- 页面风格简洁，不追求复杂视觉设计

### 预览要求

- 图片：浏览器直接展示
- 文本：文本预览页展示内容
- PDF：浏览器可预览则直接内嵌，否则下载
- 其他格式：显示“不支持预览，请下载”

---

## 8. 安全要求

1. 密码必须加密存储，禁止明文保存。
2. 所有文件访问必须经过权限校验。
3. 上传文件名必须做安全处理，防止路径穿越。
4. 分享链接应使用随机高熵 token。
5. 过期分享链接必须不可访问。
6. 对上传大小、类型、路径进行校验。
7. 对关键操作保留基础日志。

---

## 9. 非功能要求

### 可维护性

- 代码应清晰、模块化
- DTO、实体、表单对象分离
- 公共异常统一处理
- 返回结构统一

### 性能

- 列表接口支持分页（如实现成本可接受）
- 下载与预览采用流式输出，避免一次性读入大文件
- 大文件上传下载时避免明显阻塞主流程

### 测试

至少补充以下测试：

- 启动测试
- 认证流程测试
- 文件夹创建/查询测试
- 上传/下载流程测试
- 重命名/移动/删除测试
- 分享链接访问测试
